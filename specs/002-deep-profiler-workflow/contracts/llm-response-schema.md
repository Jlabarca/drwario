**Feature**: 002-deep-profiler-workflow
**Date**: 2026-03-08

---

## Current Schema (v2.0)

The LLM returns a JSON array of findings:

```json
[
  {
    "category": "CPU",
    "severity": "Warning",
    "title": "Frame drops exceeding target FPS",
    "description": "Average frame time 18.5ms with P95 spikes to 42ms.",
    "recommendation": "Profile Update() methods for expensive operations.",
    "metric": 42.0,
    "threshold": 16.67,
    "confidence": "High",
    "environmentNote": null,
    "scriptPath": "Assets/Scripts/EnemyAI.cs",
    "scriptLine": 47,
    "assetPath": null
  }
]
```

## Updated Schema (v3.0)

No structural changes to finding schema. The additions are in the **prompt input**, not the response.

### New Prompt Section: profilerMarkers

Added to the user prompt when marker data is available:

```json
{
  "profilerMarkers": [
    {
      "name": "Physics.Simulate",
      "avgInclusiveMs": 12.3,
      "avgExclusiveMs": 8.1,
      "maxInclusiveMs": 35.2,
      "avgCallCount": 1.0
    },
    {
      "name": "Rendering",
      "avgInclusiveMs": 6.8,
      "avgExclusiveMs": 3.2,
      "maxInclusiveMs": 15.4,
      "avgCallCount": 1.0
    }
  ]
}
```

### System Prompt Addition

The system prompt is updated to instruct the LLM:

> When `profilerMarkers` data is available, reference specific marker names in your findings rather than guessing which subsystem is expensive. Use the marker timing data to attribute frame budget usage.

### SSE Streaming Format (input to DrWario, not a contract we control)

**Claude SSE**:
```
event: content_block_delta
data: {"type":"content_block_delta","index":0,"delta":{"type":"text_delta","text":"..."}}
```

**OpenAI SSE**:
```
data: {"choices":[{"delta":{"content":"..."}}]}
```
```
data: [DONE]
```

DrWario accumulates `text`/`content` fields and runs progressive JSON parsing to emit findings as they complete.
